PrairieLearn's support for rich and highly customizable question types is a draw for many instructors. At the heart of this functionality are dozens of built-in elements, such as [`pl-drawing`](https://docs.prairielearn.com/pl-drawing/), which lets students assemble intricate diagrams by dragging and dropping objects. To complement `pl-drawing`, we have recently added a new element: [`pl-sketch`](https://docs.prairielearn.com/elements/pl-sketch/) provides a dedicated drawing canvas for freehand sketches of mathematical curves, as well as a wide range of grading options to allow fully automated mastery content or exams.

From a student perspective, the primary difference is the mode of drawing: while `pl-drawing` is based on drag and drop of pre-defined elements that can be rotated and resized, `pl-sketch` is designed around freehand drawing of curves with the mouse or a trackpad.

Take for example the question below, and how the choice of element determines how students construct their answer. The `pl-drawing` element requires students to pick a type of curve (straight or quadratic), drag it onto the canvas, and then adjust it until it matches the expected output. The `pl-sketch` element on the other hand expects students to draw the curve from scratch with full control over its shape.

For students, the tools that they have available to solve a question can significantly change their learning experience. For example, the `pl-drawing` version of the question above might unintentionally give hints to students by only offering them linear or quadratic curve fragments to choose from. Placing pre-defined curve segments via drag and drop is also rather different from the drawing and reasoning process and that one would apply when drawing on paper. The freehand drawing process of `pl-sketch` is more effective at testing the same skillset as a paper exam.

### Drawing aids and options

One important feature of `pl-sketch` that complements the freehand drawing capabilities is that curves can be drawn in arbitrary order or in multiple segments. In addition to some light smoothing that is immediately applied to curves, small gaps and overlaps are ignored for grading purposes, so that the entire curve is treated as a continuous function. Even in situations where emulating the freehand drawing process is less important for a question, `pl-sketch` can provide an easy to use toolbox for curve sketching. For example, the provided `spline` and `polyline` tools allow students to define reference points that are then automatically connected via straight lines or [splines](<https://en.wikipedia.org/wiki/Spline_(mathematics)>).

### Automated grading criteria

The `pl-sketch` element supports a range of different grading criteria. For grading curve sketches, comparisons to reference functions are a convenient option that can be customized further by defining comparison ranges and tolerances. Points and asymptotes can be similarly graded based on their expected number and coordinates.

Sometimes, questions might be designed to allow a range of different answers. For example, a question might require students to draw any positive, monotonically increasing function that connects two points. To support such complex requirements, `pl-sketch` element provides a collection of modular grading criteria that implement checks like _"Is the function monotonically increasing in a given x-range?"_ or _"Is the function always greater than a given value or other function?"_

Grading criteria can be customized and stacked, for example to give partial points based on different tolerance ranges, or to require prior criteria to be satisfied before another one is applied. This allows even complex questions to be fully auto-graded, enabling the use of the element for mastery learning or randomized, computer-based testing.

To make it easier for students to review their submissions and understand mistakes, each grading criterion can be set up with custom feedback to explain (or avoid revealing, if desired) why a submission received a specific percentage score. Once the correct answer has been revealed, both students and manual graders can also compare a submission to the provided sample solution by visually overlaying them.
